# ЛАБОРАТОРНАЯ №2. Инструмент контейнеризации Docker

Требования:

1) Подключиться по ssh к ВМ master-0.
   Перенести сценарий prepare.sh в ВМ и запустить его.
```
$ su -
# chmod +x ./prepare.sh
# ./prepare.sh
# exit
```
   После того как сценарий отработал, переподключиться по ssh к ВМ master-0.
   Попробовать выполнить команды от пользователя:
```
$ docker pull debian:13.3  # загрузка образа из registry

$ docker images  # вывести список образов на машине
```

   В случае успеха, в списке появится образ.

2) Ознакомиться с самыми часто используемыми командами Docker и их опциями.
   Попробовать на примере любого образа: загрузить, запустить, поставить на него пакеты, удалить, и.т.д.
   * **docker pull**
   * **docker images**
   * **docker run (-p --rm -it -d -v --entrypoint --env)**
   * **docker start**
   * **docker stop**
   * **docker rm**
   * **docker rmi (-f)**
   * **docker build**
   * **docker commit**
   * **docker cp**
   * **docker ps (-a)**
   * **docker save**
   * **docker load**
   * **docker logs (-f --tail)**
   * **docker container/volume/network prune**
   * **docker volume prune**
   * **docker network prune**
   * **docker system df**

   Подсказки по командам можно получать, например, через --help
```
$ docker <cmd_name> --help
$ docker build --help

# Поместить в background контейнер запущенный в интерактивном режиме можно комбинацией клавиш ctrl p q
```

3) Написать свой собственный **Dockerfile**, придерживаться best practice при его создании, содержимое примерно следующее:
   * базовый образ
   * установка необходимых пакетов
   * сборка своего приложения или копирование в образ скриптов с кодом
   * запуск приложения по умолчанию
   * *можете попробовать сделать multi-stage image

   Упаковать в образ можно любое приложение ( предпочтительнее взять компилируемый язык ), например REST или сценарий который выполняет некую работу и журналирует это в stdout/stderr.
   Полученный образ должен запускаться и работать, в запущенном контейнере можно смотреть лог его работы или делать к нему запросы и получать ответы.

4) Написать **docker-compose.yaml** манифест, запустить с помощью docker compose как минимум 2 образа.
   Собрать минимум 2 образа и запустить их через docker compose, также показать их работу.
   Можно взять за основу образы из предыдущего пункта или сделать новый.

   Команды которые могут быть полезны для взаимодействия с docker compose:
   * **docker compose up -d**
   * **docker compose down**
   * **docker compose restart**
   * **docker compose stop**
   * **docker logs <container_name>**

## При показе выполненного задания
   * Продемонстрировать использование базовых команд docker, docker compose ( запустить, остановить, посмотреть лог и.т.д. ).
   * Уметь объяснить содержимое своего Dockerfile ( как поставить пакет, скопировать файл и.т.д. ).
   * Уметь объяснить содержимое своего docker-compose.yaml файла.
